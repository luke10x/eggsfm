# ingamefm

A C++17 header-only library for real-time FM music and sound effects in games.  
Synthesises audio from patch data using the **ymfm** YM2612 emulator — no audio files needed.

---

## The idea

Traditional game audio loads `.ogg`, `.wav` or `.mp3` files. ingamefm synthesises everything from FM patch parameters at runtime, the same way a Sega Genesis produced sound in 1990. The result is:

- **Zero audio assets** — music and SFX are pure numbers in your source code.
- **Tiny footprint** — a full song is a few hundred bytes of text.
- **Fully procedural** — tempo, pitch and timbre can be changed at any time in code.
- **Independent volumes** — music and SFX are generated by two separate chip instances and mixed with their own scalars, so neither affects the other.

The workflow: design patches and songs in [Furnace tracker](https://github.com/tildearrow/furnace) (free, open source), copy the pattern text directly into your C++ source, assign patches to instrument IDs, and call `player.start()`. The companion VST/AUv3 plugin **ARM2612** can also export patches directly in ingamefm format from inside your DAW.

## Build

### Native — Linux / macOS

```bash
YMFM=/path/to/ymfm/src
SDL_INC=/path/to/SDL2/include
SDL_LIB=/path/to/libSDL2.a   # or $(sdl2-config --libs)

g++ -std=c++17 -O2 \
    -I$YMFM -I$SDL_INC \
    $YMFM/ymfm_misc.cpp $YMFM/ymfm_adpcm.cpp \
    $YMFM/ymfm_ssg.cpp  $YMFM/ymfm_opn.cpp \
    demo5.cpp \
    $SDL_LIB \
    -o demo5 && ./demo5
```

On macOS add the system frameworks:

```bash
    -framework Cocoa -framework IOKit -framework CoreVideo \
    -framework CoreAudio -framework AudioToolbox \
    -framework ForceFeedback -framework Carbon \
    -framework Metal -framework GameController -framework CoreHaptics \
    -lobjc
```

### Emscripten — Web (demo3, demo4, demo6)

These demos require ImGui and WebGL2. The exact flags are in the build block at the bottom of each `.cpp` file. The short version:

```bash
YMFM=/path/to/ymfm/src
IMGUI=/path/to/imgui

em++ -std=c++17 -O2 \
    -I$YMFM -I$IMGUI \
    $YMFM/ymfm_misc.cpp $YMFM/ymfm_adpcm.cpp \
    $YMFM/ymfm_ssg.cpp  $YMFM/ymfm_opn.cpp \
    $IMGUI/imgui.cpp $IMGUI/imgui_draw.cpp \
    $IMGUI/imgui_tables.cpp $IMGUI/imgui_widgets.cpp \
    $IMGUI/backends/imgui_impl_sdl2.cpp \
    $IMGUI/backends/imgui_impl_opengl3.cpp \
    demo6.cpp \
    -s USE_SDL=2 -s FULL_ES3=1 \
    -s MIN_WEBGL_VERSION=2 -s MAX_WEBGL_VERSION=2 \
    -s ALLOW_MEMORY_GROWTH=1 -s ASYNCIFY \
    --shell-file shell4.html \
    -o demo6.html

python3 -m http.server   # → http://localhost:8000/demo6.html
```

**Note on buffer size:** the Web Audio API requires a buffer of at least 256 samples. The demos handle this automatically with `#ifdef __EMSCRIPTEN__`. If you write your own Emscripten build, use `desired.samples = 256` rather than `128`.

## Quick start — music only

```cpp
#include "ingamefm.h"

static constexpr YM2612Patch MY_BASS = {
    .ALG = 4, .FB = 5, .AMS = 0, .FMS = 0,
    .op = {
        { .DT=3,.MUL=1,.TL=34,.RS=0,.AR=31,.AM=0,.DR=10,.SR=6,.SL=4,.RR=7,.SSG=0 },
        { .DT=0,.MUL=2,.TL=18,.RS=1,.AR=25,.AM=0,.DR=12,.SR=5,.SL=5,.RR=6,.SSG=0 },
        { .DT=0,.MUL=1,.TL= 0,.RS=0,.AR=31,.AM=0,.DR= 6,.SR=3,.SL=6,.RR=5,.SSG=0 },
        { .DT=0,.MUL=1,.TL= 0,.RS=0,.AR=31,.AM=0,.DR= 7,.SR=2,.SL=5,.RR=5,.SSG=0 },
    }
};

// Paste directly from Furnace: Edit → Copy Pattern Data
static const char* SONG =
"org.tildearrow.furnace - Pattern Data (8)\n"
"8\n"
"E-2007F|.......\n"
"G-2007F|.......\n"
"A-2007F|.......\n"
"OFF....|.......\n"
"E-2007F|.......\n"
"G-2007F|.......\n"
"A-2007F|.......\n"
"OFF....|.......\n";

int main(int, char**) {
    SDL_Init(SDL_INIT_AUDIO);

    IngameFMPlayer player;
    player.set_song(SONG, /*tick_rate=*/60, /*speed=*/6);
    player.add_patch(0x00, MY_BASS);   // instrument 00 in pattern → MY_BASS
    player.play();                     // blocks until finished

    SDL_Quit();
}
```

## Non-blocking playback in a game loop

```cpp
SDL_AudioSpec desired{};
desired.freq     = IngameFMPlayer::SAMPLE_RATE;  // 44100
desired.format   = AUDIO_S16SYS;
desired.channels = 2;
desired.samples  = 128;    // ~3 ms latency; use 256 for Emscripten
desired.callback = IngameFMPlayer::s_audio_callback;
desired.userdata = &player;

SDL_AudioSpec obtained{};
SDL_AudioDeviceID dev = SDL_OpenAudioDevice(nullptr, 0, &desired, &obtained, 0);

player.set_song(SONG, 60, 6);
player.add_patch(0x00, MY_BASS);
player.start(dev, /*loop=*/true);
SDL_PauseAudioDevice(dev, 0);

while (running) {
    SDL_Event e;
    while (SDL_PollEvent(&e)) { /* ... */ }
    SDL_WaitEventTimeout(nullptr, 1);
}

SDL_CloseAudioDevice(dev);
```

**Tip:** prefer `SDL_WaitEventTimeout` over `SDL_Delay(1)`. On most systems `SDL_Delay` sleeps 10–15 ms regardless of the argument, adding noticeable SFX latency. `SDL_WaitEventTimeout` returns immediately when any event arrives.

## Channels

The YM2612 has 6 channels split across two ports:

| Channel | Port | Hardware channel |
|---|---|---|
| 0, 1, 2 | Port 0 | 0, 1, 2 |
| 3, 4, 5 | Port 1 | 0, 1, 2 |

`IngameFMChip` handles port routing transparently — pass any channel number 0–5 to `load_patch`, `set_frequency`, `key_on`, and `key_off`. `IngameFMPlayer` supports all 6 channels (`MAX_CHANNELS = 6`).

---

## Two-chip architecture

ingamefm runs two independent `IngameFMChip` instances internally:

**`ym_music_`** runs the Furnace song on all music channels (0 to `MAX_CHANNELS-1`). It is completely unaware of SFX and is never interrupted or modified by the SFX system.

**`ym_sfx_`** is a voice pool of `sfx_voices_` independent channels (0 to N-1). These channels are entirely separate from the music channels — no shared register space, no shared state. SFX grab whichever voice is free or has the lowest priority, play there, and release when done.

Each chip generates independently into its own buffer each audio callback, then the two buffers are mixed in floating point with separate volume scalars before writing to SDL output. The `ym_sfx_` chip is only mixed in when at least one SFX voice is active, preventing idle-chip hum.

This means:
- Music plays completely uninterrupted regardless of SFX activity.
- SFX never steal or silence music channels.
- Music volume and SFX volume are completely independent.
- Timbre is preserved at all volume levels (volume is a post-synthesis float multiply, not TL register manipulation).

## Sound effects

### Setup

```cpp
// How many voices the SFX pool has (default 3, max MAX_SFX_VOICES = 6).
// Call before start().
player.sfx_set_voices(3);

// Pre-parse an SFX pattern. May be called at any time, even during playback.
// id: 0-255. Pattern is a single-channel Furnace pattern, same format as songs.
player.sfx_define(0, SFX_JUMP_PATTERN, /*tick_rate=*/60, /*speed=*/3);
player.sfx_define(1, SFX_COIN_PATTERN, 60, 3);
```

### Triggering

```cpp
// Must be called with the SDL audio device locked.
SDL_LockAudioDevice(dev);
player.sfx_play(/*id=*/0, /*priority=*/4, /*duration_rows=*/10);
SDL_UnlockAudioDevice(dev);
```

`sfx_play` fires immediately — the first note key-on happens before the function returns, so the sound starts within the next audio buffer (~3 ms at `samples=128`).

### Priority rules

- The player scans voice slots right-to-left: prefers a free slot, then the lowest-priority active slot.
- If all slots have equal or higher priority than the incoming SFX, the call is silently ignored.
- When an SFX expires its slot is released automatically.
- Use priority 1 or higher for SFX.

### Suggested priority scale

| Priority | Use |
|---|---|
| 1–2 | Ambient / background (footsteps, breathing, ambient loops) |
| 3–4 | Common gameplay (jump, coin, attack, dialogue) |
| 5–6 | Significant events (damage, fanfare, launch, scream) |
| 7–9 | Critical / must-hear (death, level-up, boss) |

### SFX patterns

An SFX pattern is a standard single-channel Furnace pattern. Use a fast row rate (speed=3, ~50 ms/row) for responsive game feel:

```cpp
static const char* SFX_JUMP =
"5\n"
"C-4007F\n"   // note, instrument 00, volume 7F
"G-4007F\n"
"C-5007F\n"
"OFF....\n"
".......\n";

player.sfx_define(SFX_ID_JUMP, SFX_JUMP, 60, 3);
```

Pass a `duration_rows` to `sfx_play` long enough to cover the full pattern.

## Volume control

```cpp
player.set_music_volume(0.7f);   // 70%
player.set_sfx_volume(1.0f);     // 100% (default)
```

Both are thread-safe atomic stores — call from any thread at any time without locking the audio device. Scalars are applied as float multipliers on each chip's output buffer after synthesis, preserving timbre at all volume levels.

---

## Pattern format

ingamefm parses the text produced by Furnace's **Edit → Copy Pattern Data**. Each row has N channel columns separated by `|`. Each column is 7 characters:

```
A-4 0 0 7F
│   │ │ └── Volume (hex, 00-7F, or .. to inherit)
│   │ └──── Instrument ID (hex, 00-FF, or .. to inherit)
│   └─────── Note (C-0 .. B-9, OFF, or ... to hold)
└─────────── Note name + accidental + octave
```

Effects columns beyond the 7-character minimum are silently ignored.

### Row rate formula

```
seconds_per_row = speed / tick_rate
```

Common values:

| tick_rate | speed | ms/row | Feel |
|---|---|---|---|
| 60 | 3 | 50 | Very fast, arcade |
| 60 | 6 | 100 | Fast, action game |
| 60 | 12 | 200 | Medium, RPG |
| 60 | 20 | 333 | Slow, ambient |

## Playing notes directly

`chip()` returns the music chip (`ym_music_`) for manual note triggers — useful for instrument previews or the keyboard piano in demo2/demo3. Always lock the SDL audio device first.

```cpp
SDL_LockAudioDevice(dev);
player.chip()->load_patch(PATCH_GUITAR, 2);
player.chip()->set_frequency(2, IngameFMChip::midi_to_hz(60));  // C4
player.chip()->key_on(2);
SDL_UnlockAudioDevice(dev);

// Release later
SDL_LockAudioDevice(dev);
player.chip()->key_off(2);
SDL_UnlockAudioDevice(dev);
```

`chip()` returns null before `start()` or `play()` is called.

## Patch format reference

```cpp
struct YM2612Operator {
    int DT;   // Detune:        -3..+3
    int MUL;  // Multiplier:    0-15
    int TL;   // Total Level:   0-127  (0 = loudest, 127 = silent)
    int RS;   // Rate Scaling:  0-3
    int AR;   // Attack Rate:   0-31   (31 = fastest)
    int AM;   // AM Enable:     0-1
    int DR;   // Decay Rate:    0-31
    int SR;   // Sustain Rate:  0-31
    int SL;   // Sustain Level: 0-15
    int RR;   // Release Rate:  0-15
    int SSG;  // SSG-EG:        0=off, 1-8=hardware modes 0-7
};

struct YM2612Patch {
    int ALG;              // Algorithm:      0-7
    int FB;               // Feedback:       0-7  (OP1 only)
    int AMS;              // AM Sensitivity: 0-3
    int FMS;              // FM Sensitivity: 0-7
    YM2612Operator op[4]; // OP1, OP2, OP3, OP4
};
```

**TL** is attenuation, not level — 0 is loudest, 127 is silent. On carriers, TL controls output volume. On modulators, TL controls modulation depth.

**AR=31** is the fastest possible attack (sample-accurate onset). All built-in SFX patches use AR=31 on every operator.

**ALG** determines which operators are carriers vs modulators:

| ALG | Carriers |
|---|---|
| 0–3 | OP4 only |
| 4 | OP2, OP4 |
| 5–6 | OP2, OP3, OP4 |
| 7 | OP1, OP2, OP3, OP4 (all carry, no FM modulation) |

## API summary

### `IngameFMPlayer`

```cpp
// ── Song setup ────────────────────────────────────────────────────────────
void set_song(const std::string& pattern_text, int tick_rate, int speed);
void add_patch(int instrument_id, const YM2612Patch& patch);

// ── SFX setup ─────────────────────────────────────────────────────────────
void sfx_set_voices(int n);    // 1..MAX_SFX_VOICES (default 3). Call before start().
void sfx_define(int id, const std::string& pattern, int tick_rate, int speed);

// ── Playback ──────────────────────────────────────────────────────────────
void play(bool loop = false);                  // blocking
void start(SDL_AudioDeviceID, bool loop);      // non-blocking
void stop(SDL_AudioDeviceID);
bool is_finished() const;

// ── SFX  (call with audio device locked) ──────────────────────────────────
void sfx_play(int id, int priority, int duration_rows);

// ── Volume  (thread-safe, no lock needed) ─────────────────────────────────
void set_music_volume(float v);   // 0.0 (silent) .. 1.0 (full)
void set_sfx_volume(float v);

// ── Direct chip access  (call with audio device locked) ───────────────────
IngameFMChip* chip();    // returns ym_music_; null before start()/play()

// ── SDL audio wiring ──────────────────────────────────────────────────────
static void s_audio_callback(void* userdata, Uint8* stream, int len);
static constexpr int SAMPLE_RATE    = 44100;
static constexpr int MAX_CHANNELS   = 6;   // music channels on ym_music_
static constexpr int MAX_SFX_VOICES = 6;   // max voice pool size on ym_sfx_
```

### `IngameFMChip`

```cpp
void load_patch(const YM2612Patch& patch, int ch);     // ch 0-5
void set_frequency(int ch, double hz, int octaveOffset = 0);
void key_on(int ch);
void key_off(int ch);
void enable_lfo(bool enable, uint8_t freq = 0);         // freq 0-7
void generate(int16_t* stereo_buffer, int samples);
void write(uint8_t port, uint8_t reg, uint8_t val);     // raw register access

static double midi_to_hz(int midiNote);   // A4 = MIDI 69 = 440 Hz
static constexpr uint32_t YM_CLOCK = 7670453;
```

## Error handling

`set_song()` and `sfx_define()` throw `std::runtime_error` with a message and line number if the pattern text is malformed:

```cpp
try {
    player.set_song(SONG, 60, 6);
    player.add_patch(0x00, PATCH_FLUTE);
    player.sfx_set_voices(3);
    player.sfx_define(0, SFX_JUMP, 60, 3);
    player.start(dev, true);
} catch (const std::exception& e) {
    fprintf(stderr, "ingamefm setup failed: %s\n", e.what());
    return 1;
}
```
